ASPECTO EVALUADO DIAGRAMA UML
"Aparece una narrativa para cada caso de uso, en la que se describe claramente como mínimo los siguientes aspectos:

• Nombre del caso de uso
• Propósito
• Actores que participan
• Cómo se activa
• La descripción paso a paso de la interacción entre el sistema y el actor durante la ejecución del caso de uso (especificando lo que hace el actor y lo que hace el sistema, solamente lo visible por el actor)
• El lenguaje utilizado es entendible para el cliente del sistema.

ASPECTO EVALUADO SOFTWARE
- El diseño de interfaces corresponde al análisis y modelado realizado,
- Muestran una imagen por cada una de las pantallas que mostrará el sistema,
- Cada interfaz indica todo lo que verá el usuario durante su interacción,
- "Se toman en cuenta todas las restricciones (seguridad, calidad y manejo de errores)",
- "El software cumple con todos los requerimientos, sin fallas ni problemas".
